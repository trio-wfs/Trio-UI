Button

Buttons communicate actions that users can take. They are typically placed throughout your UI, in places like dialogs, modal windows, forms, cards, and toolbars. Buttons express what action will occur when the user interacts with them.

info Overview

psychology Purpose
Buttons trigger immediate actions or navigate users to new views. They are the primary mechanism for user interaction in forms, dialogs, and workflows.
priority_high Visual Hierarchy
Use contained buttons for primary actions, outlined for secondary actions, and text buttons for tertiary or low-priority actions.
accessibility Accessibility
All buttons include proper focus states, keyboard navigation, and ARIA labels. Minimum touch target is 32px for accessibility compliance.

rule Usage Guidelines

check_circle When to Use
  • Submitting forms (Save, Update, Submit)
  • Confirming or canceling actions in dialogs
  • Navigating to a new page or view
  • Starting a workflow or process
  • Adding or creating new items
  • Destructive actions with clear labeling (Delete, Remove)
cancel When NOT to Use
  • Navigation between views (use tabs or links instead)
  • Displaying a list of options (use select or menu)
  • Toggling between states (use switch or checkbox)
  • Opening a dropdown menu (use select component)
  • More than 2 buttons in a small space (prioritize actions)
  • Generic "Click Here" labels (be specific about action)

recommend Best Practices

check_circle Do
  • Use specific, action-oriented labels (Save, Delete, Create)
  • Limit to 1 primary button per section or dialog
  • Place primary action on the right in button groups
  • Use icons only when they reinforce the action meaning
  • Maintain consistent button sizing throughout forms
  • Use Primary blue only for Save/Update actions
cancel Don't
  • Use vague labels like "OK", "Click Here", or "Submit"
  • Place more than 3 buttons in a single button group
  • Use all caps text (use sentence case)
  • Mix button sizes in the same row or group
  • Use icons without labels in primary actions
  • Use Primary blue for Edit actions (use Secondary)

palette Color Semantics

Button colors communicate meaning and action consequence. Follow these semantic rules consistently across the application.

Color When to Use Example Actions Visual
primary Save or Update operations only Save, Update, Apply Changes
secondary Edit actions and general navigation Edit, Cancel, Close, Back
success Positive confirmation actions Confirm, Approve, Activate
error Destructive or irreversible actions Delete, Remove, Deactivate
warning Actions requiring caution Archive, Deprecate, Force Update
info Informational or help actions Learn More, View Details, Help
