ContentContainer

The standard paper surface for TRIO WFS page content. White background (#FFFFFF), 1px border (#E0E0E0), 4px radius. No shadows — ever. Use for every card, panel, and content section on a page.

category Atomic Component verified Production Ready

info Overview

ContentContainer is the building block for all page content. It provides a consistent white surface with a gray header bar, 1px border, and 4px radius. Containers can nest inside other containers for grouped content sections.

title With Title

Pass a title prop to render the gray header bar (48px, #F5F5F5) with Subtitle2 typography. Optionally pass titleActions to render controls (icons, menus) on the right side of the header.

crop_free Without Title

Omit title to render a borderless surface with only padding — no header bar. Use when the context makes the container's purpose self-evident.

fullscreen Flush Content (padding: 0)

Set padding={0} for content that fills edge-to-edge, like AG Grid. The title bar (if present) still renders normally — only the body padding is removed.

layers Flush Border Radius

Set flush={true} to remove border radius. Use when containers stack directly against each other or against a page edge and rounded corners would create visible gaps.

account_tree Nested Containers

ContentContainers can nest inside each other for grouped sub-sections. Use 16px gap between siblings inside a parent container's padded body.

recommend Best Practices

check_circle Do
  • Every paper section on a page is a ContentContainer — hand-rolled paper cards (white box + border + arbitrary header) are not allowed. Use the component so the title strip stays consistent across the system.
  • The title strip uses background.default (#F5F5F5) — persistent across every container. 48px tall, 16px left / 8px right padding, body Subtitle2.
  • Containers can nest — KV row lists inside a container body, or a smaller container inside a larger one. The title strip is required at the top level of each container.
  • Wrap AG Grid in a ContentContainer when the grid lives inside a tab; standalone grid pages skip the wrapper because PHT and grid share a continuous border.
  • Use flush radius when stacking containers that should read as one continuous surface (e.g. grid + PHT)
  • Title actions slot (right-aligned): chips, icon buttons, small text buttons — not primary CTAs (those belong in the PHT)
cancel Don't
  • Don't wrap standalone components in ContentContainer — Alert, MetricCard, and Stepper already have their own structure. The container rule is for grouping cards / sections (Profile, Compliance, Bill Rates), not for chrome around components that own their own surface.
  • Don't hand-roll paper cards — arbitrary white-box-with-border patterns drift over time. Use ContentContainer so the title strip, border, radius, and padding match the rest of the system.
  • Don't add drop shadows to ContentContainer — shadows are only for floating overlays (modals, menus, popovers). Paper surfaces use borders, not elevation.
  • Don't use nested cards instead of internal borders — sub-groups inside a container separate with 1px #E0E0E0 horizontal borders, not nested paper cards.
  • Don't use background.default (#F5F5F5) for the container body — that's the page canvas color and the title strip color, never the body. The body is always background.paper.
  • Don't put the page's primary CTA in the title actions slot — the primary action lives in the PHT's singleButtonContent slot, not in a container's title strip
