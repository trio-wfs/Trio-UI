Button Group

Groups related button actions together to show multiple options in a single, connected unit. Use to present related actions that should be visually grouped.

category Atomic Component
link Figma: buttonGroup (2172:9605)
check_circle Production Ready

Overview

info Purpose
Button groups connect related actions to show they belong together. Use when:
  • Multiple actions are related and should be visually connected
  • Actions modify the same object or data
  • You need to save horizontal space for multiple buttons
tune Variants
Variant: contained, outline
Size: sm (32px), md (38px)
Color: primary, secondary
Orientation: horizontal, vertical
Buttons: 2-6 buttons (button-3 to button-8)
accessibility Accessibility
  • Each button is individually focusable via keyboard
  • Use clear, descriptive labels for each button
  • Avoid icon-only buttons without text labels
  • Proper ARIA roles handled by MUI ButtonGroup

rule Usage Guidelines

check_circle When to Use
  • Group related actions that modify the same object
  • Use 2-4 buttons for optimal usability
  • Use clear, action-oriented labels
  • Use primary color only for confirming/save actions
  • Use secondary for editing/prompt actions
cancel When NOT to Use
  • Don't group unrelated actions together
  • Don't exceed 6 buttons (Figma limit: button-8)
  • Don't use for exclusive selection (use Toggle Button)
  • Don't mix primary and secondary colors in one group
  • Don't use for navigation (use Tabs instead)

Size Variants

Small (32px) - Toolbars and dense interfaces

Color Variants

Primary blue is for confirming/save actions. Secondary is for editing/prompt actions.

Primary - Confirming Actions
Secondary - Editing/Prompt Actions

Orientation

Button groups can be arranged horizontally (default) or vertically for narrow layouts.

Horizontal (Default)
Vertical - For narrow containers
