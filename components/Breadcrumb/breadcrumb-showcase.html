Breadcrumb

Navigation path strip showing the user's location within a hierarchy. Sits between the Page Header Toolbar and the main content area. Supports an optional record count and action buttons on the right.

category Atomic Component verified Production Ready

info Overview

psychology Purpose
Trail of links indicating the user's location in a nested hierarchy. Always anchored at the top of content, below the page header.
priority_high Visual Hierarchy
Low — Breadcrumb is auxiliary navigation. Provides context but is not the primary way users move through the app.
accessibility Accessibility
Wrap in <nav aria-label="breadcrumb"> with an ordered list. The current page is the last item and not a link; it carries aria-current="page".

rule Usage Guidelines

check_circle When to Use
Correct
  • Showing hierarchical navigation within the application
  • Pages that are 2+ levels deep in the information architecture
  • List or table views where a record count is helpful
  • Pages with filter/export/reset toolbar actions
cancel When NOT to Use
Incorrect
  • Don't exceed 6-7 path segments — truncate middle items with ellipsis
  • Don't use for step-by-step wizard navigation — use a Stepper
  • Don't put non-navigation actions in the breadcrumb strip
  • Don't use on top-level pages with no parent hierarchy

widgets Variants

Two divider styles — breadcrumb uses /, Links uses |. Use breadcrumb as the default; Links for tab-style navigation.

recommend Best Practices

check_circle Do
Correct
  • Breadcrumb in state="Links" is the grid view switcher — the All | Active | Pending | … chrome at the top of standalone grid pages is NOT the Tabs component. It's this. Same 40px height, same background.secondary, pipe dividers, optional inline count per item.
  • On a grid page, Breadcrumb-Links lives in the PHT's breadcrumbContent slot (full variant) — PHT and grid share one continuous border, no gap.
  • Selected item in state="Links" uses a 2px primary-blue bottom border indicator on the item itself (not on the strip), with text.primary color. Non-selected items use primary-blue link text. Hover underlines.
  • Optional inline counts go after each label in text.secondary. The selected item's count flips to primary blue.
  • Always mark exactly one item as selected — the current page or view
  • Keep link labels short — use the page/section name only
  • Use the breadcrumb state (slash dividers) for navigation hierarchy; use Links (pipe dividers) for view switching
  • Show record count when the page is a list or table view
cancel Don't
Incorrect
  • Don't use the Tabs component for grid view switching — that's Pattern C in the §11 taxonomy, and it's Breadcrumb-Links, not Tabs.
  • Don't use state="Links" for breadcrumb navigation — pipe dividers signal "lateral views of the same level," not hierarchical path. Use the breadcrumb state for path navigation.
  • Don't restyle the selected-item indicator — 2px primary-blue bottom border on the item, item text in text.primary. Changing the indicator position or color breaks the visual rhyme with Tabs (which use top indicator instead).
  • Don't exceed 6-7 path segments — truncate middle items with ellipsis
  • Don't put non-navigation actions in the breadcrumb strip
  • Don't use Breadcrumb for step-by-step wizard navigation — use a Stepper
  • Don't use single-letter or ambiguous labels
