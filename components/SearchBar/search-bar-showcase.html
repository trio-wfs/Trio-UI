Search Bar

A text input with an attached search icon button — used to filter lists, grids, or page content.

category Custom Component menu_book Storybook design_services Zeplin verified Production Ready

info Overview

psychology Purpose
SearchBar wraps a TextField with one or two support button segments containing a search icon. The button visually attaches to the input, forming a single compound control.
priority_high Behavior
Pressing Enter or clicking the search button triggers the onSearch callback. Three type variants control button placement: left, right, or both.
accessibility Accessibility
Input has proper ARIA labels. Focus ring uses the standard blue outline. Disabled state prevents interaction on both input and button.

rule Usage Guidelines

check_circle When to Use
Correct
  • Use the Medium size by default
  • Reserve Small for dense toolbars (e.g., inline grid filters)
  • Reserve Large for primary page-level search
  • Provide a clear placeholder that tells the user what is searchable
cancel When NOT to Use
Incorrect
Using SearchBar for navigation
  • Don't use SearchBar for navigation or routing — it's a filter control
  • Don't use the Both type unless there's a clear reason for dual triggers
  • Don't use for global/site-wide search — use the App Bar search field instead

recommend Best Practices

check_circle Do
Correct
  • Use the Medium size by default. Reserve Small for dense toolbars and Large for primary page-level search.
  • Provide a clear placeholder that tells the user what is searchable: "Search workers, shifts, or orders..." is better than just "Search".
cancel Don't
Incorrect
  • Don't use SearchBar for navigation or routing. It's a filter control, not a site-wide search.
  • Don't use the Both type unless there's a clear reason for dual triggers. Left is the standard pattern across TRIO WFS.
