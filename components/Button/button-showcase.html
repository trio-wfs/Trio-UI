Button

Triggers actions or navigation. Variant + color expresses weight and consequence — contained primary for commits, secondary for workhorse actions, text for utilitarian, error contained for permanent destructive.

category Atomic Component verified Production Ready

rule Variant Roles

Each variant + color combination has a specific role. Choosing the right one communicates the weight and consequence of an action.

Button Role When to Use
Primary commit action The "done" button. Save, Submit, Create, Confirm. Completes a flow or permanently changes state. Limit to one per page or modal. Medium size in modals/forms, small when nested in components.
Workhorse action The default choice. Dropdowns, modal triggers, panel openers, downloads, exports. Actions without permanent consequences. Dominant in header toolbar button groups at medium size. If you need a button and aren't sure which, start here.
Utility action Actionable but low-consequence operations: downloading a document, opening a secondary flow, filtering. Outlined primary gives it enough visual weight to signal "this does something" without competing with the commit button.
Utilitarian / Cancel Low-emphasis page actions: open/close panels, expand/collapse accordions, Show More, Add Section. Pair with caret or plus icons. Also the correct choice for Cancel — always pair a primary text cancel with a primary contained submit.
Permanent destructive action Delete Permanently, Remove, Revoke Access. Use sparingly — the red should make someone pause. Reserve for actions that cannot be undone.
Soft destructive action Remove from list, Unassign, Clear. The action is reversible or lower stakes than a permanent delete. Outlined reduces visual urgency while keeping the semantic warning.
Positive-reinforcement submit Approve, Activate, Publish. Final actions where the outcome is explicitly positive — approving a qualified worker for a shift, activating a configuration. Reinforces "this is a good thing."
Consequential but reversible Archive, Deactivate, Force Sync. The user should think before proceeding, but the action can be undone. A step below error-red in severity.

recommend Best Practices

check_circle Do
Correct
  • One primary contained button per page or modal — this is the commit action
  • Pair text "Cancel" with primary contained submit — never contained cancel
  • Place primary action on the right in button groups
  • Use secondary outlined as your default utility button
  • Use semantic colors (error, success) sparingly to signal consequence
  • Medium size in modals/forms, small in tables and nested components
cancel Don't
Incorrect
  • Use multiple primary contained buttons on the same page
  • Use secondary contained for Cancel — use text variant instead
  • Use secondary contained as a submit or completion mechanism
  • Use primary blue for Edit actions — use secondary outlined
  • Mix button sizes in the same row or group
  • Use icons without labels on primary commit actions

palette Color Semantics

Button colors communicate the weight and consequence of an action. Primary blue is reserved for commit actions. Semantic colors signal risk or reward. Secondary is the neutral default.

Color Consequence Level When to Use Contained Outlined
primary Permanent — commits state Save, Submit, Create, Update. One per page/modal. The action that completes a flow.
secondary None — workhorse action Contained: the default button — dropdowns, exports, filters, modal triggers, toolbar groups. Text: cancel, panel toggles, accordions.
—
primary outlined None — utility action Low-consequence but actionable: downloading a file, opening a secondary flow, filtering. Enough visual weight to signal action without competing with the commit button. —
error Permanent — destructive Contained: permanent deletion, revoking access. Outlined: reversible removal (unassign, clear, remove from list).
success Permanent — positive outcome Final actions with an explicitly positive result. Approve a worker placement, activate a configuration, publish a schedule. Reinforces "this is a good thing."
warning Reversible — needs thought Consequential but undoable. Archive, deactivate, force sync. The user should pause, but the action isn't permanent.
info None — informational Help, Learn More, View Details. Rarely used as a standalone action — typically appears in contextual help or onboarding.
