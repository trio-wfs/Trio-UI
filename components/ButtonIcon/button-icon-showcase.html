Button Icon

Icon-only buttons for compact actions where the icon alone communicates the intent. Use sparingly — prefer labeled buttons when the action isn't immediately obvious.

category Atomic Component menu_book Storybook design_services Zeplin verified Production Ready

info Overview

psychology Purpose
ButtonIcon wraps MUI IconButton with TRIO WFS design tokens. Use when space is constrained and the icon is unambiguous — toolbars, toggle groups, and inline table actions.
tune Variants
Two visual styles: contained (filled background) and ghost (transparent, with hover ripple). Each supports primary and secondary color schemes, plus sm and md sizes.
accessibility Accessibility
Always provide an aria-label for accessibility. Supports a badge overlay for notification counts. Pair with Tooltip for discoverability.

rule Usage Guidelines

check_circle When to Use
Correct
  • Compact toolbars and action bars
  • Inline table row actions (edit, delete, more)
  • Toggle groups using ToggleButton strip
  • When the icon is universally recognized (edit, delete, close)
  • Notification indicators with badge overlays
cancel When NOT to Use
Incorrect
  • Don't use icon-only buttons for primary page actions — use a labeled Button
  • Don't use ambiguous icons without a tooltip
  • Don't mix contained and ghost styles in the same toolbar strip
  • Don't use badge counts above 99 — truncate to "99+"

recommend Best Practices

check_circle Do
Correct
  • Always provide an aria-label
  • Use ghost for inline/table actions — least disruptive
  • Use contained primary for toolbar toggles and high-visibility actions
  • Use contained secondary as the default toolbar button
  • Group related icon buttons into a toggle strip using ToggleButton
cancel Don't
Incorrect
  • Don't use icon-only buttons for primary page actions — use a labeled Button instead
  • Don't use ambiguous icons without a tooltip
  • Don't mix contained and ghost styles in the same toolbar strip
  • Don't use badge counts above 99 — truncate to "99+"
