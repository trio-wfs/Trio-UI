Date Picker

Date and time selection inputs built on MUI X DatePicker, themed to TRIO WFS tokens.

info Overview

The DatePicker lets users select a date (or date + time) via a text input that opens a calendar popup. It wraps MUI X DatePicker with TRIO design tokens for consistent styling across the platform.

When to use: Shift scheduling, assignment dates, credential expiry, timecard entries, report date ranges.

Implementation: Uses @mui/x-date-pickers with dayjs adapter. The calendar popup is themed to match the Figma spec — 36px day cells, #2196F3 selection, outlined today indicator.

widgets Variants

Date

Standard date-only picker. Opens a 320px calendar popup.

Date + Time

Date + time picker with hour/minute/period columns alongside the calendar.

recommend Best Practices

DO
  • Use type="date" for date-only fields (most common)
  • Use type="dateTime" only when time is required (shift start/end)
  • Set minDate / maxDate to prevent invalid selections
  • Always include a label for accessibility
  • Show helper text for date format expectations
DON'T
  • Don't use dateTime when only a date is needed
  • Don't leave date pickers without min/max when there's a valid range
  • Don't use a plain text field for date input — always use DatePicker
  • Don't set unreasonably wide date ranges (e.g., 100 years back)

integration_instructions Code Example

import { DatePicker } from '@trio-wfs/ui';
import dayjs from 'dayjs';

// Date only
<DatePicker
  label="Start Date"
  value={startDate}
  onChange={setStartDate}
/>

// Date + Time
<DatePicker
  type="dateTime"
  label="Shift Start"
  value={shiftStart}
  onChange={setShiftStart}
/>

// With constraints
<DatePicker
  label="Assignment Date"
  minDate={dayjs('2024-01-01')}
  maxDate={dayjs('2024-12-31')}
  value={date}
  onChange={setDate}
  helperText="Must be within 2024"
/>
