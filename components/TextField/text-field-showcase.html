TextField

Text fields let users enter and edit text. They typically appear in forms and dialogs. Text fields provide a single-line or multi-line input for freeform data entry.

category Atomic Component link Figma: text-field verified Production Ready

info Overview

psychology Purpose
Text fields enable users to input freeform text data. They're used for short responses (single-line) or longer content (multi-line) in forms and data entry interfaces.
priority_high States
Text fields have clear visual states: default (unfocused), focus (active input), error (validation failed), and disabled (not editable).
accessibility Accessibility
All text fields include proper labels, ARIA attributes, keyboard navigation, and clear focus indicators for screen reader support.

rule Usage Guidelines

check_circle When to Use
  • Collecting freeform user input in forms
  • Short text entries (name, email, phone)
  • Longer text content (descriptions, comments)
  • Search queries and filter inputs
  • Editable data fields in tables
  • User-generated content entry
cancel When NOT to Use
  • Selecting from predefined options (use Select)
  • Yes/no questions (use Checkbox or Switch)
  • Choosing one option from many (use Radio)
  • Date/time input (use Date Picker)
  • Large blocks of formatted text (use rich text editor)
  • Password fields without proper security

recommend Best Practices

check_circle Do
  • Always include a clear, descriptive label
  • Use helper text for additional context
  • Show error messages clearly and specifically
  • Keep single-line inputs for short data
  • Provide clear placeholder examples
  • Use appropriate input types (email, tel, etc.)
cancel Don't
  • Use placeholder as the only label
  • Make required fields unclear
  • Show generic error messages
  • Use multi-line for short inputs
  • Disable without explanation
  • Make inputs too narrow for expected content
