Alert

Alerts communicate important messages to users — informational, success, warning, or error — with optional actions and dismiss controls. In clinical workflows, alert severity directly communicates patient safety risk.

category Atomic Component verified Production Ready

info Overview

psychology Purpose
Communicates the status of a view, section, or operation — success, warning, error, info. Inline element placed above or within the content it describes.
priority_high Visual Hierarchy
Medium-high when present — Alert draws attention but does not block. Reserved for messages that warrant interruption but don't require a modal.
accessibility Accessibility
Native role="alert" (error/warning) or role="status" (info/success). Screen readers announce on render. Dismissible alerts include an accessible close button.

  • Color cohesion — every colored element inside the alert uses the same semantic color. Border, icon, title, description, and action buttons all share the one color that matches the alert's severity. Don't mix the semantic color with text.primary / text.secondary for the title and description.
  • Semantic color choice is functional, not stylistic — info for neutral callouts, success for positive outcomes, warning for caution, error for failures or destructive actions.
  • For content users must not miss, use a full info-color border around the container — not a left-accent-only treatment. Reserve for compliance warnings and critical requirements, not general emphasis.
  • Drop shadows are only for floating overlays (Modals, Menus, Popovers) — never on inline alerts.
  • Use error only for blocking issues that require immediate action
  • Keep titles short — 4 words or fewer
  • Pair every alert with a clear action or dismiss path
  • Use standard variant as the default — least visually disruptive
  • Auto-dismiss success alerts after 3-5 seconds when possible
  • Include an icon that reinforces the severity level

palette Color Semantics

Alert colors communicate urgency and action consequence. Follow these semantic rules consistently across the application.

Color When to Use Healthcare Context Visual
default (Blue) Informational messages. No action required. System status, tips, guidance
error (Red) Something failed or is blocked. Requires user action. Failed submission, expired credential, compliance block
warning (Orange) Potential issue — non-blocking. User should be aware. Expiring credentials, approaching limits
success (Green) Positive confirmation. Action completed successfully. Shift confirmed, credential approved, submission accepted
