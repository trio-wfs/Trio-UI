Stepper

A horizontal progress bar that shows position within a multi-step workflow, with completed, active, and upcoming states per step.

category Custom Component menu_book Storybook design_services Zeplin verified Production Ready

info Overview

psychology Purpose
Use the Stepper when a task requires completing a defined sequence of steps — onboarding flows, multi-page wizards, or staged configuration workflows. It keeps users oriented.
priority_high Key Rules
Always show all steps upfront — do not reveal steps progressively. Keep step labels short (1–3 words). Desktop-first — the bar spans the full page width.
accessibility Accessibility
Each step has ARIA labels indicating its state (completed, current, upcoming). Connectors are decorative. Steps use role="listitem" within a role="list" container.

rule Usage Guidelines

check_circle When to Use
Correct
1
2
3
  • Show all steps upfront — never hide upcoming steps
  • Keep step labels to 1–3 words so they don't wrap
  • Place the stepper at the top of the page, above the form content
  • Use a consistent step count throughout a single flow
cancel When NOT to Use
Incorrect
8+ steps or non-linear flows
  • Don't use more than 7 steps — consider consolidating
  • Don't use the stepper for non-linear or optional flows
  • Don't make steps clickable to jump backwards without validation
  • Don't use for single-page forms — the overhead isn't warranted
