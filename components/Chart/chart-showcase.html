info Overview

psychology Purpose
The Chart component wraps AG Charts with TRIO's extended 10-color palette, Roboto typography, and Material-style icon SVGs applied via the AG Charts theming API.
priority_high Usage
Engineers import <Chart options={...} /> — no theme configuration required. Use for fill rate trends, order volume, worker status distributions, and more.
accessibility Accessibility
Color vision simulation bar lets you preview all charts under 7 different color blindness conditions. The 10-color palette was chosen for maximum distinguishability.

rule Usage Guidelines

check_circle When to Use
Correct
Bar for comparisons, Line for trends, Donut for proportions
  • Use <Chart> — never raw <AgCharts> — so the theme is always applied
  • Keep chart titles short and meaningful (14–16px, medium weight)
  • Use a legend when 3 or more series are present
  • Use bar/column for comparisons, line/area for trends, donut for proportions
  • Keep y-axis labels concise — use K/M suffixes for large numbers
cancel When NOT to Use
Incorrect
Custom colors or pie charts
  • Don't set custom colors inline — use the TRIO palette from trioChartPalette
  • Don't use more than 5–6 series in one chart — it becomes unreadable
  • Don't place charts inside a modal (900px large modal only if absolutely necessary)
  • Don't use pie charts — use donut instead (center label adds context)
  • Don't mix chart types for the same data type in the same view

widgets Variants

All charts below are rendered live by AG Charts v13 using trioAgChartsTheme. Hover to see tooltips.

Color Vision

Bar — Fill Rate by Department

Stacked or grouped bars for comparing metrics across departments or time periods.

Line — Order Volume Over Time

Trends over time. Use for weekly/monthly volumes, submission rates, or SLA performance.

Donut — Worker Status Distribution

Proportional breakdown. Use for status distributions, credentialing stages, or category splits.

Area — Shift Coverage Trend

Filled area for volume trends where cumulative context matters.

Scatter — Bill Rate vs Fill Rate

Correlation between two numeric variables. Use to compare agency pricing against performance.

Bubble — Rate / Fill Rate / Volume

Three-variable scatter. Bubble size encodes a third metric — here, submission volume.

Combination — Submissions + Fill Rate

Bar + line on dual axes. Use when two related metrics have different scales.

stacked_bar_chart Multi-Series — Agency Comparison

Multiple series pull from the palette in order. This example shows 5 agencies across 6 weeks using colors 1–5.

touch_app Interaction & Layout Patterns

Stacking, labelling, interpolation, and crosshair — all Community tier. Hover any chart to explore interactions.

Stacked Bar — Filled / Pending / Unfilled

Use when the total and the composition both matter.

Normalized Stacked — Proportional View

Same data normalized to 100%. Use when comparing proportions regardless of volume.

Bar — Data Labels

Values rendered directly on bars. Use on dense dashboards where reading from the axis adds friction.

Stacked Area — Agency Submission Volume

Multiple filled areas stacked. Use to show individual contribution alongside the running total.

Smooth Interpolation

Curved line between data points. Use for biological or coverage trends.

Step Interpolation

Flat-then-jump between points. Use for scheduling or rate changes.

Crosshair — Axis Tracking

Vertical and horizontal crosshair lines snap to the nearest data point on hover.

align_horizontal_left Horizontal & Progress Bars

Standard bar series with direction: 'horizontal' and width: 10. Long category labels read naturally left-to-right.

Horizontal Bar — Fill Rate by Department

Use when category labels are long or when ranking order is the primary story.

Horizontal Stacked — Filled / Pending / Unfilled

Same composition pattern rotated. Each row is a department's total order count.

Progress Fill — Credentialing Completion

Two-series stacked horizontal: actual value + remaining. Each bar uses a palette color.

palette Color Semantics

10-color extended palette for TRIO WFS data visualization. Colors are assigned in order — series 1 always gets color 1, series 2 gets color 2, and so on. If there are more series than colors, the palette repeats from the beginning.
