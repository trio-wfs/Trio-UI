Metric Card

Metric Cards display numeric KPIs and data summaries in a compact 150px-tall card. They support 1–4 metric values, four footer styles, and per-metric semantic color overrides. Cards are always used in horizontal groups of 2 or more, filling available width with 16px gaps between them.

category Custom Component verified Production Ready

info Overview

bar_chart Purpose
Display compact numeric KPIs at a glance. Each card shows a label, 1–4 metric values with sub-labels, and an optional footer (progress bar, icons, or labels). Height is always exactly 150px regardless of variant.
palette Color System
Uses the info tone scale (900 → 700 → 500 → 300) as the primary data viz color. Semantic colors (error, warning, success) can override any metric starting at 500, ascending to 700, then 900 for additional urgency.
grid_view Layout Rules
Always used in groups of 2+ cards in a horizontal flex row with 16px gaps. Cards have a minimum width of 170px and expand to fill available space equally. Labels truncate with ellipsis at minimum width.

rule Usage Guidelines

check_circle When to Use
Correct
  • Dashboard KPI summaries (counts, totals, statuses)
  • 2+ related metrics that benefit from side-by-side comparison
  • Proportional data that a progress bar meaningfully represents
  • Operational metrics with urgency states (error/warning/success)
  • Grouped comparison: 1 primary metric + 2 supporting sub-metrics
  • Lists of 4 related values that don't need chart visualization
cancel When NOT to Use
Incorrect
Single standalone card without context
  • Single standalone card — always pair with at least one other card
  • More than 4 metric values — use a data table instead
  • Trend over time — use a chart component
  • Detailed breakdowns — use a table or expandable list
  • Non-numeric content — use a summary card pattern instead
  • Mixing info and semantic colors randomly — use per-metric overrides intentionally

widgets Metric Count Variants

The number of metrics in the array drives the layout. 1–3 values display as equal-width columns. 4 values switch to a vertical list. Value font size scales down to fit (34px → 24px → 16px) as more columns are added.

1 Metric — Single Column (fs=34, info/900, bar footer)
2 Metrics — Two Equal Columns (fs=34, info/900 + info/700)
3 Metrics — Three Equal Columns (fs=24, info/900 + 700 + 500)
4 Metrics — Vertical List (fs=16, list layout, no footer)

view_headline Footer Variants

The footer area below the metric columns is swappable. The bar is the default. Icons and labels can replace the bar. Any footer can be omitted entirely.

footer="bar" — Proportional segmented progress bar (default)
footer="icons" — Per-metric Material icon indicator
footer="labels" — Secondary caption labels (Figma: style=custom)
footer="none" — No footer, metrics fill available height

view_sidebar Grouped Layout

The grouped layout places one large primary metric on the left (34px) and stacks 2 smaller caption/value pairs on the right (16px). This is the Figma "groupe/bar" variant. Supports bar or no footer.

layout="grouped" with bar footer

toggle_on States

Any metric can be assigned a semantic color override (error, warning, success) for urgency signaling. Semantic colors start at 500 for the first occurrence and ascend to 700, then 900. Info metrics retain their positional tone assignment. You can mix info and semantic colors within the same card.

All metrics → error (500 → 700 → 900)
Mixed — info (1st metric) + error override (2nd metric)

label Label Icon Variants

The label row supports an optional left icon (Material Icons only) and an optional help_outline icon on the right. Both are optional and can be combined freely.
