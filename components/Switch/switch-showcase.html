Switch

Toggle between two states (on/off). Use for immediate actions that take effect instantly without requiring a save or submit action.

category Form Control verified Production Ready

rule Usage Guidelines

check_circle When to Use
Correct
  • Binary on/off toggles that take effect immediately
  • Enabling or disabling features (e.g., notifications, auto-assignment)
  • Toggling settings within a preferences panel
  • Activating/deactivating a filter or view mode
  • Any reversible state change that doesn't require confirmation
cancel When NOT to Use
Incorrect
  • Selecting from multiple options — use Radio Group instead
  • Agreeing to terms or consent — use Checkbox instead
  • Changes that require a save/submit action — use Checkbox instead
  • Triggering an action (e.g., "Submit") — use Button instead
  • Irreversible actions that need confirmation — use Button + Modal instead

recommend Best Practices

check_circle Do
Correct
Clear label describes what the switch controls
  • Always provide a clear, descriptive label
  • Use for settings that apply immediately
  • Position labels consistently across a settings panel
  • Make the current state obvious through color and position
  • Group related switches together in settings views
cancel Don't
Incorrect
No label — user cannot tell what this controls
  • Don't use a switch without a visible label
  • Don't use vague labels like "Toggle" or "Option"
  • Don't use for actions that require save/submit confirmation
  • Don't use for irreversible or destructive actions
  • Don't mix switch and checkbox patterns in the same form
