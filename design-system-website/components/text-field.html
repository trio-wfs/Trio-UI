TextField

Text fields allow users to enter and edit text. They support single-line and multi-line input with various states including default, error, focus, and disabled.

Input Production Ready

Usage

Use text fields when you need users to input short text, emails, passwords, or longer content like descriptions and comments. TextFields adapt to single-line or multi-line contexts and provide clear feedback through states.

When to Use

  • Single-line: For names, emails, search queries, and short responses
  • Multi-line: For descriptions, comments, notes, and longer text content
  • With icons: To provide context (search icon, validation checkmark)
  • With help text: To clarify requirements or validation rules

When NOT to Use

  • Don't use for binary choices — use Checkbox or Radio instead
  • Don't use for selecting from predefined options — use Select or Autocomplete
  • Avoid overloading with too many optional features (icons, chips, help text)

Props

All props are optional. Default values match Figma specification.

Prop Type Default Description
type 'single-line' | 'multi-line' 'single-line' Input mode: single line (height 36px) or multi-line textarea (height 90px)
state 'default' | 'error' | 'focus' 'default' Visual state affecting border color: default (#9E9E9E), error (#DB4537), focus (#64B5F6)
disabled 'no' | 'yes' 'no' Disable input and show disabled styling (#EEEEEE background, #9E9E9E text)
label boolean true Show/hide label above input
labelText string 'Label' Text content for label
helpText boolean false Show/hide help text below input
helperText string 'Help text' Text content for help text (color changes to error in error state)
placeholderLabel string 'Placeholder' Placeholder text shown when input is empty
inputContent string '' Initial value for input (use value prop in controlled components)
iconRight boolean false Show icon on right side of input (e.g., validation checkmark)
adormentInput boolean false Show icon/adornment on left side of input (e.g., search icon)
iconSupport boolean false Enable support icon (typically near help text)
inputFill boolean false Use filled background (#FAFAFA) instead of transparent
chipContent boolean false Show chip(s) below input for added content/tags
value string undefined Controlled component value (use with onChange)
onChange (value: string) => void undefined Callback when input value changes

Code Examples

Basic Single-line Input

React/TypeScript 
import { TextField } from '@ahtg/components';

export function LoginForm() {
  const [email, setEmail] = useState('');

  return (
    <TextField
      type="single-line"
      label={true}
      labelText="Email Address"
      placeholderLabel="you@example.com"
      value={email}
      onChange={setEmail}
      helpText={true}
      helperText="We'll never share your email"
    />
  );
}

Multi-line with Error State

React/TypeScript 
import { TextField } from '@ahtg/components';

export function FeedbackForm() {
  const [feedback, setFeedback] = useState('');
  const isRequired = feedback.length === 0;

  return (
    <TextField
      type="multi-line"
      label={true}
      labelText="Feedback"
      placeholderLabel="Tell us what you think..."
      value={feedback}
      onChange={setFeedback}
      state={isRequired ? 'error' : 'default'}
      helpText={isRequired}
      helperText="Feedback is required"
    />
  );
}

Search Input with Icon

React/TypeScript 
import { TextField } from '@ahtg/components';

export function SearchBar() {
  const [query, setQuery] = useState('');

  return (
    <TextField
      type="single-line"
      adormentInput={true}
      placeholderLabel="Search patients..."
      value={query}
      onChange={setQuery}
      label={false}
    />
  );
}

Accessibility

Keyboard Interaction

  • Tab: Focus input field
  • Shift+Tab: Focus previous interactive element
  • Type: Enter text content
  • Backspace: Delete text
  • Ctrl+A: Select all text

Screen Reader

  • Label is associated with input via <label> element
  • Help text is announced as additional context
  • Error state is indicated by aria-invalid and aria-describedby
  • Placeholder text is read after label and help text

WCAG Compliance

  • ✓ WCAG 2.1 Level AA compliant
  • ✓ Color contrast meets 4.5:1 ratio for text
  • ✓ Focus indicators visible for keyboard navigation
  • ✓ Label properly associated with input
