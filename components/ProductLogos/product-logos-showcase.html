Product Logos

The TRIO product mark in three forms — full wordmark for light backgrounds, full wordmark for dark backgrounds, and the standalone hexagon icon. Use the right variant for the surface so the mark reads cleanly.

category Brand Asset verified Production Ready

info Overview

psychology Purpose
Identifies the surface as TRIO — in the header bar, on login and empty states, in marketing footers, and anywhere brand presence matters. Three locked variants ensure the mark always renders correctly on any background.
priority_high Visual Hierarchy
High — logos are recognition anchors, not decoration. Reserve them for moments where the user benefits from knowing this is TRIO. Don't sprinkle logos across the UI as filler.
accessibility Accessibility
Defaults to alt="TRIO". Override alt when the logo is acting as a link (e.g., alt="TRIO home"). Pair with aria-label on the wrapping anchor when the logo is the only interactive content.

rule Usage Guidelines

check_circle When to Use
Correct
  • The primary header bar / app shell — full wordmark, dark or white per the bar's background
  • Login, signup, and password reset pages — mark anchors the page
  • Empty states for the whole app (not for individual sections within it)
  • Marketing footers, PDF report headers, exported documents
  • Favicons, app icons, mobile launchers — use logoOnly for icon-sized contexts
cancel When NOT to Use
Incorrect
  • Decoratively scattered across the UI — logos aren't ornaments
  • As an in-product button or icon — use ButtonIcon or a Material icon
  • White logo on a light background or dark logo on a dark background — mark disappears
  • Within form fields, table cells, or other dense data — reads as visual noise
  • Recolored or restyled — the mark is locked. Never change colors, proportions, or spacing

straighten Sizes

Width drives sizing — height derives from the natural aspect ratio unless explicitly set. Use the natural width whenever the layout permits; scale down only for header bars or compact contexts. Don't scale up beyond natural size — the mark loses crispness.

recommend Best Practices

check_circle Do
Correct
Dark logo on light surface, sized for the header
  • Pick the variant that contrasts with the background — dark on light, white on dark
  • Set width only — let height derive from the natural aspect ratio
  • Use logoOnly for icon-sized contexts (favicons, mobile launchers, tabs)
  • Preserve clear space around the mark — at least the height of the hexagon on all sides
  • Override alt with action context when the logo links somewhere ("TRIO home")
cancel Don't
Incorrect
  • Don't place the dark logo on a colored brand background — readability collapses
  • Don't recolor, restroke, rotate, or skew the logo — the mark is locked
  • Don't scale the wordmark below ~100px width — legibility suffers; switch to logoOnly
  • Don't stretch by setting incompatible width and height together
  • Don't use the logo as a decorative pattern, watermark, or background fill
