Autocomplete

Autocomplete inputs extend Text Field with typeahead search and dropdown selection. Supports single-select and multi-select (chip) modes. Used throughout AHTG for searching providers, agencies, facilities, and other VMS entities.

category Form Component link Figma: 2381:6441 verified Production Ready

infoOverview

Types

Single — one value selected, replaces input text on selection.
Multi — multiple values shown as removable chips inside the input.

States

Default — gray border (#9E9E9E).
Focus — blue border (#64B5F6).
Error — red border (#DB4537) with helper text.
Disabled — muted border + fill.

Accessibility

ARIA combobox role. Keyboard navigation: Arrow keys to browse options, Enter to select, Escape to close, Backspace to remove last chip (multi mode).

recommendBest Practices

check_circle Do
  • Use single for one-value selections (assigned provider, primary facility)
  • Use multi for many-value selections (resource pools, agencies, tiers)
  • Always provide a label — never rely on placeholder alone
  • Show provider counts in options for VMS context (e.g. "Agency Name (12 providers)")
  • Disable the field when the context makes it inapplicable — pair with helper text explaining why
  • Use for dynamic lists with 5+ options that benefit from search filtering
cancel Don't
  • Don't use Autocomplete for short static lists (<5 items) — use Select instead
  • Don't use placeholder text as a substitute for a visible label
  • Don't mix single and multi behavior for the same field type across the app
  • Don't disable without communicating why (silent disabled state is confusing)
  • Don't use for boolean or binary choices — use Checkbox or Switch instead
  • Don't allow free-form text entry unless the field explicitly supports it
