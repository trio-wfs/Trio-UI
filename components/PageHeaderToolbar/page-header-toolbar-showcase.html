Page Header Toolbar

The top-of-page toolbar used across all TRIO WFS list and detail views. Contains the page title, optional chips and eyebrow, right-side actions, and an optional Breadcrumb strip.

category Custom Component verified Production Ready

info Overview

psychology Purpose
Persistent header bar at the top of a page with title, filters, search, and primary actions. Sets the context for everything below it.
priority_high Visual Hierarchy
Medium-high — orients the user and exposes the section's primary actions. Always above the main content.
accessibility Accessibility
Use <header> semantics. Primary action button must be reachable via keyboard tab. Avoid keyboard traps in filter controls.

rule Usage Guidelines

check_circle When to Use
Correct
Title + Actions + Breadcrumb on every list/detail page
  • Use full as the default for list/grid views — breadcrumb + chips provide important context
  • Use default for simple utility pages without deep navigation
  • Keep page title to the entity name — avoid verbs (use "Shift Scheduler", not "Manage Shifts")
  • Put the primary action in ButtonGroup, secondary in SingleButton
  • Use indicator only when the entity has a meaningful status affecting the whole page
cancel When NOT to Use
Incorrect
Multiple toolbars or destructive actions in toolbar
  • Don't put more than 4 chips in the title row — truncate with "+N More"
  • Don't use more than one toolbar per page
  • Don't put destructive actions (Delete, Archive) in the toolbar — move to a confirmation modal flow
  • Don't omit eyebrow text on complex pages — it provides orientation for new users

recommend Best Practices

check_circle Do
  • Use the right variant for the page type: full for most pages (title, breadcrumb, actions); default when no breadcrumb is needed; NewCanvas for the Programs surface and other transparent/canvas pages.
  • Standalone grid pages use the full variant with the breadcrumbContent slot filled by the Breadcrumb component in state="Links" — that's the grid view switcher (All | Active | Pending | …). The PHT and grid share one continuous border, no gap.
  • Actions slot composition is canonical: inputTextFieldContent takes a TextField or SearchBar (scoped search); buttonGroupContent takes one or more ButtonGroups (connected secondary actions like the icon trio Filter / Columns / More); singleButtonContent takes the single primary Button for the page (e.g. "+ New Worker").
  • Place the PHT inset 16px from the canvas edges — it sits inside the page canvas's padding, not flush to the browser edge. The future full-width variant for DW / AI layouts is the only exception.
  • Detail page section nav lives at the content wrapper level, not in the PHT — use the Tabs component (Pattern A) flush at the top of the #FAFAFA wrapper.
  • Below lg (1200px), the PHT title row and action row stack vertically — chips and eyebrow stay inline with the title.
  • Keep page title to the entity name — avoid verbs (use "Shift Scheduler", not "Manage Shifts")
  • Use the indicator strip only when the entity has a meaningful status affecting the whole page
  • Title-to-eyebrow gap is 4px (spacing.xs) — come from the component, don't override at the call site
cancel Don't
  • Don't pour loose Button or IconButton instances into the actions area — group them via ButtonGroup so they share the segmented chrome the rest of the system uses.
  • Don't put the primary action inside a ButtonGroup of secondary actions — the primary lives in singleButtonContent as its own Button.
  • Don't use the default variant on grid pages — you lose the breadcrumbContent slot the view switcher needs to live in. Use full.
  • Don't put the Tabs component in the PHT for detail page section nav — that's a wrapper-level Tabs (Pattern A). The only tab-like content in the PHT is Breadcrumb-Links for grid view switching.
  • Don't run the PHT to the browser edge on standard pages — the page canvas keeps a 16px gutter. Full-width is reserved for the planned DW / AI layout variant.
  • Don't invent a new header layout — always use a PHT variant.
  • Don't put more than 4 chips in the title row — truncate with "+N More"
  • Don't use more than one toolbar per page
  • Don't put destructive actions (Delete, Archive) in the toolbar — move them into a confirmation modal flow
  • Don't disable buttons in the actions area when a page has no actions — remove the action group entirely
