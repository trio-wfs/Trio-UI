Button

Triggers an action or event such as saving data, opening a dialog, or navigating to another page. Buttons communicate what action will occur when a user interacts with them.

Overview
When to use
check_circle Use a button when
  • A user needs to take an action — save, submit, confirm, or navigate
  • The action is the primary or secondary focus of the current context
  • You need to clearly communicate what happens when the user clicks
  • An action needs a visual affordance that stands out from surrounding content
cancel Do not use a button when
  • The action is inline within a sentence — use a text link instead
  • You need to navigate to an external URL in a paragraph context
  • The button would compete with a primary action on the same surface
  • You need more than three buttons in a single action group

Usage
Do and don't
✓ Do

Use one contained button per context. Pair it with an outlined or text button for secondary actions. This creates a clear visual hierarchy.

✗ Don't

Don't use multiple contained buttons side by side. Every button competing for attention means none of them stand out.

✓ Do

Use action-oriented labels that describe exactly what happens. Pair icons with labels for clarity on primary actions.

✗ Don't

Don't use vague labels like "Click here", "OK", or "Submit" without context. Labels should describe the outcome, not the gesture.

Prop Type Default Description
variant 'contained' | 'outlined' | 'text' | 'destructive' | 'neutral' 'contained' Controls visual emphasis and intent. Use contained for primary actions only.
size 'md' | 'sm' 'md' Controls button height and font size. md is default for most contexts.
startIcon ReactNode — Icon placed before the label. Use Material Icons only.
endIcon ReactNode — Icon placed after the label. Useful for directional actions like Export.
disabled boolean false Prevents interaction. Renders at 38% opacity. Do not use to hide unavailable actions — remove them instead.
onClick () => void — Handler called when the button is clicked.
