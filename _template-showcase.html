Section 6: Best Practices — Icon: recommend. Content rule (read carefully): This section is about technique — given you've chosen this component, how do you use it well?
  • Do — imperative authoring guidance about specific choices: labels, focus order, accessibility, sizing, content, edge cases. Format as verb-led commands. Example: "Use specific verb labels — 'Reject submission' not 'OK'", "Restore focus to the trigger element on close".
  • Don't — technique-level anti-patterns. Specific authoring mistakes, not scenario mismatches. Example: "Don't auto-focus the destructive button — Cancel must be the default". If the Don't names a different component as the alternative, it belongs in Usage Guidelines instead.
Voice: imperative ("Use…", "Restore…", "Don't…") — never noun phrases. Same card-illustration pattern as Usage Guidelines when a visual makes the rule tangible.
