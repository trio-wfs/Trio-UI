Slider

Continuous-range selector for numeric values where position communicates more than the exact number. Use for thresholds, ranges, and tuning interactions where users want to "feel" the value.

category Form Control verified Production Ready

info Overview

psychology Purpose
Position-based numeric selection along a continuous track. Differs from NumberField by emphasizing relative position (close to min, near max) over the precise number.
priority_high Visual Hierarchy
Medium — the filled track is a primary-blue line that draws the eye, so reserve Slider for values worth highlighting (alert thresholds, fill-rate goals, scoring weights).
accessibility Accessibility
Thumb supports ArrowLeft/ArrowRight for step, Home/End for min/max. Pair with a visible label and consider valueLabelDisplay="on" when the exact value matters.

rule Usage Guidelines

check_circle When to Use
Correct
  • Threshold tuning where users want to feel the value (overtime alert hours, fill-rate target %)
  • Range selection — min and max as a paired control (date range, rate window)
  • Settings that benefit from visual feedback as the value changes
  • Values from a clear bounded range where position is meaningful
  • Coarse adjustments that don't need exact precision — users tune by feel
cancel When NOT to Use
Incorrect
  • Discrete integer entry where the value is the point — use NumberField
  • Values where users will type the exact number — use TextField
  • Selection from a known list of choices — use Select or RadioGroup
  • Binary on/off settings — use Switch
  • Currency, dates, or any high-precision input — use TextField with validation

recommend Best Practices

check_circle Do
Correct
Labeled marks + auto value bubble make the abstract scale concrete
  • Use labeled marks when the scale is conceptual (Low / Med / High) rather than literal
  • Pair with a visible value display (label, helper text, or valueLabelDisplay="auto") when exact values matter
  • Match Slider size to surrounding form controls — medium with medium inputs, small with small
  • Use range sliders (two thumbs) when the value is a range, not when users pick one of two endpoints
  • Set step to match the precision users care about — not finer than they can perceive
cancel Don't
Incorrect
No label, no marks, no value display — user can't tell what 50 means
  • Don't ship a Slider with no label and no value display — users can't tell what they're choosing
  • Don't use Slider when precise numeric entry matters — users will fight the thumb position
  • Don't use a 0-to-100 Slider for two-state choices — use Switch or ToggleButton
  • Don't combine a Slider and a NumberField that bind the same value — pick one input method
  • Don't use semantic colors (success/warning/error) on the track — reserve primary blue for it (§5)
