Navigation Vertical

The primary left-sidebar navigation used across all TRIO WFS desktop views. Supports open/closed states, a title header with optional dropdown selector, expandable sub-items, settings sections, and a secondary slide-out panel.

category Custom Component verified Production Ready

info Overview

psychology Purpose
NavigationVertical is the persistent left-rail navigation. It holds primary section links with icons, supports expandable sub-navigation, an optional settings section, and a title header that can act as a program/context selector dropdown.
priority_high Layout
Open state is 260px with icon + label + chevron. Closed state collapses to a 72px icon rail with tooltips. A hamburger toggle switches between states. Title and subtitle appear above the nav items.
swap_horiz Title Dropdown
The title can be a static header or a dropdown selector using the Menu component. Use this to let users switch between programs, facilities, or other top-level contexts without leaving the current view.
Full navigation — interactive: click the hamburger to toggle open ↔ closed

widgets Variants

The nav has two primary states controlled by the hamburger toggle button.

Open (260px) — icons + labels
Closed (72px) — icon rail with tooltips

Title Header

The title area sits above the nav items, separated by a divider. It supports three modes: static text (no interaction), clickable (custom onTitleClick handler), or dropdown selector (pass titleMenuItems to open a Menu on click). The subtitle provides secondary context like a facility name.

Static title
Dropdown title (chevron indicator)

Expandable Sub-Items

Items with expandable: true show a 16px chevron on the right. Clicking toggles the chevron direction and reveals sub-items indented 40px. Selected sub-item text uses primary.dark.

Collapsed
Expanded with selection

Settings & SubSection Panel

When settings is enabled, a settings section appears at the bottom. With subSection enabled, settings items appear in a 260px secondary panel that slides out to the right, triggered by a "Program Settings" button. Without subSection, settings items render inline with a left-border selected style.

recommend Best Practices

check_circle Do
  • Always include a left icon for primary nav items — icons are critical for scanability in time-pressured clinical environments
  • Keep primary nav labels short — 1 or 2 words maximum. The 260px width is tight with an icon, label, and chevron
  • Auto-expand the active parent item on page load so users see where they are immediately
  • Use the title dropdown to let users switch programs without navigating away — update the nav items accordingly when the selection changes
  • Show the subtitle for context (e.g. facility name) so users know which program scope they're in
cancel Don't
  • Don't apply a background color to the nav — it is always transparent against the page
  • Don't use settings-style items (left border) for primary navigation — the two patterns are visually distinct for a reason
  • Don't nest sub-navigation more than one level deep — if you need deeper hierarchy, reconsider the IA
  • Don't put more than 8-10 primary items in the nav — consider grouping or a secondary navigation pattern
  • Don't mix static and dropdown titles within the same app — pick one pattern and be consistent
