Toggle Button

A grouped strip of 3–8 icon buttons sharing a single primary-color border. Used in toolbars and breadcrumb bars for compact view or action switching.

category Atomic Component verified Production Ready

info Overview

psychology Purpose
Persistent on/off button (often used in groups) for view modes, filter toggles, or formatting controls. Visually shows its active state until toggled off.
priority_high Visual Hierarchy
Medium — ToggleButton is a settings-level control. Carries equal weight to other inline controls in its group.
accessibility Accessibility
Use aria-pressed to communicate the pressed state. In groups, each button retains independent focus; arrow keys are not required unless the group is exclusive (then convert to RadioGroup).

rule Usage Guidelines

check_circle When to Use
Correct
  • Use 3 buttons as the default — filter, export, reset is the standard TRIO WFS pattern
  • Always mark one slot as active when used for view switching
  • Use sm in the breadcrumb bar and compact toolbars
  • Provide aria-label on every slot
cancel When NOT to Use
Incorrect
6+ buttons in a single strip
  • Don't exceed 5 buttons — use a Menu or overflow pattern instead
  • Don't mix ToggleButton with standalone ButtonIcon in the same toolbar strip
  • Don't use for unrelated actions — slots should share a common theme

recommend Best Practices

check_circle Do
Correct
  • Heights: sm uses 24px (tokens.controls.ghostHeight) — the same Figma-sanctioned compact exception as ButtonIcon's ghost-small. md uses tokens.controls.height.medium (38px). Confirmed against Figma node 6950:485.
  • Icons are @mui/icons-material Outlined at 16px — never another library, never another size.
  • Provide aria-label on every slot — ToggleButton is icon-only, so screen-reader labels are required.
  • Use 3 buttons as the default — filter, export, reset is the standard TRIO WFS pattern
  • Always mark one slot as active when used for view switching
  • Use sm in the breadcrumb bar and compact toolbars where 24px is the right density
cancel Don't
Incorrect
  • Don't hardcode height: 24 or height: 38 — pull from tokens.controls.ghostHeight / tokens.controls.height.
  • Don't mix ToggleButton with standalone ButtonIcon in the same toolbar strip — group through one component so the segmented chrome is consistent.
  • Don't exceed 5 buttons — use a Menu or overflow pattern instead
  • Don't use for unrelated actions — slots should share a common theme (view modes, filters, chart types)
