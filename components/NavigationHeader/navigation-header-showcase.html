Navigation Header

The top-level application header for all TRIO WFS pages. Contains the brand bar with search and user info, plus the horizontal navigation bar below.

info Overview

The NavigationHeader is a two-row component that sits at the very top of every TRIO WFS page. Row 1 is the brand bar (blue-grey) with the TRIO logo, environment badge, search, and user avatar. Row 2 is the white navigation bar with horizontal menu items.

recommend Best Practices

check_circle Do
  • NavigationHeader is fixed at the very top of every TRIO page — always present, always the same, no variants.
  • Maintain 16px padding-bottom between NavigationHeader and the page content below — that gap is canonical, not optional.
  • Show keyboard_arrow_down on nav items with dropdowns; items without dropdowns have no arrow
  • Use a 2px bottom border to indicate the single active nav item
  • Use the existing SearchBar component with type="Right" in the brand bar
  • Hide the environment badge in production, or set it to a neutral label
cancel Don't
  • Don't reuse the brand bar background color (#607D8B) — this is the only place that color appears in TRIO; it's reserved for this surface.
  • Don't introduce a responsive collapse or overflow menu — there is no responsive variant yet. Treat narrow viewports as degraded, not designed. (Paired with NavigationVertical's still-TBD responsive behavior.)
  • Don't allow more than one nav item to look active simultaneously
  • Don't omit NavigationHeader on TRIO product pages — the only exception is Digital Workers and future AI layouts
  • Don't display "TRIO WIP" or "TRIO PROD" badges in production
