engineering Engineering

Everything TRIO engineering needs to consume the design system — components, tokens, install code, theme, page rules, and reference prototypes.

lightbulb
The prototype IS the spec.
Every prototype is built with the same React components, design tokens, and page rules engineering will use in TRIO. To rebuild a screen in TRIO, you open the prototype source, lift the JSX, and swap the import from @trio-wfs/ui to whatever your project imports from. No translation layer, no separate "spec doc" to interpret.

terminalCode examples

Wire-up snippets — install command, theme provider, import shape, token-driven styling.

download Install 
# Option A — via npm (once published to GitHub Packages)
npm install @trio-wfs/ui

# Option B — local link, for active iteration alongside Trio-UI
git clone https://github.com/jszygiel1981-ship-it/Trio-UI.git
# in your project's package.json:
"@trio-wfs/ui": "file:../Trio-UI"
code Import 
import { Button, TextField, Modal, tokens, trioTheme } from '@trio-wfs/ui';
import { Box, Stack, Typography } from '@mui/material';  // layout primitives only
import CloseIcon from '@mui/icons-material/Close';          // icons OK

// Wrap your app once with the theme:
<ThemeProvider theme={trioTheme}>
  <App />
</ThemeProvider>
rule Token-driven styling — never hardcode 
// WRONG — hardcoded values
<Box sx={{ padding: '16px', color: '#2196F3' }} />

// RIGHT — token-sourced
<Box sx={{ padding: tokens.spacing.md, color: tokens.colors.primary.main }} />

handshakeWhat we need from engineering

Three things from the engineering side to keep this working.

1
Current MUI theme shape (Patrick)
Send the current TRIO theme override file. The goal is to converge tokens.ts with the production theme so the DS and TRIO agree on values, breakpoints, and component defaults. Once aligned, devs only worry about layout.
2
Two-way feedback when you lift from a prototype
When you rebuild a screen in TRIO using the prototype as a spec, flag anything you had to change — missing prop, wrong token, layout that didn't translate. Those become DS work items.
3
Storybook decision
The April plan was for Storybook to eventually run off TRIO's real codebase components. Confirm timing — do we keep the DS Storybook as the reference, or wait for the TRIO one and retire this?
