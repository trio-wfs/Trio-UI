Search Bar

A text input with an attached search icon button — used to filter lists, grids, or page content.

info Overview

SearchBar wraps a TextField with one or two support button segments containing a search icon. The button visually attaches to the input, forming a single compound control. Pressing Enter or clicking the search button triggers the onSearch callback.

view_week Type

The type prop controls which side(s) have the support button.

Search buttons on both sides. Use when the search context could be triggered from either direction.

thumb_up Best Practices

✓ Do

Use the Medium size by default. Reserve Small for dense toolbars (e.g., inline grid filters) and Large for primary page-level search.

✗ Don't

Don't use SearchBar for navigation or routing. It's a filter control, not a site-wide search. For global search, use the App Bar search field.

✓ Do

Provide a clear placeholder that tells the user what is searchable: "Search workers, shifts, or orders…" is better than just "Search".

✗ Don't

Don't use the Both type unless there's a clear reason for dual triggers. Left is the standard pattern across TRIO WFS.
