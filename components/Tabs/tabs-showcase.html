Tabs

Horizontal navigation that switches between related content panels on the same page. Use tabs when users need to switch between a small number of related views without leaving the page.

category Navigation Component menu_book Storybook design_services Zeplin verified Production Ready

info Overview

psychology Purpose
Tabs organize content into discrete sections. The active tab is indicated by a 2px primary-blue bar at the top and a medium-weight blue label. Use for parallel content sections where only one is visible at a time.
tune Anatomy
Each tab is 40px tall with 16px horizontal padding. Active state uses primary blue label (500 weight) with a 2px top indicator. Inactive tabs use secondary gray (#757575). Supports optional left/right icons at 18px.
accessibility Accessibility
Keyboard accessible with Arrow keys to navigate between tabs. Uses ARIA role="tablist" and role="tab" semantics. Disabled tabs show not-allowed cursor and are excluded from focus order.

rule Usage Guidelines

check_circle When to Use
Correct
  • Parallel content sections where only one is visible at a time
  • Worker profile sections, order detail views
  • When users need to compare related but separate data sets
  • Keep tab count between 2 and 8
  • Use scroll variants when tab count is variable
cancel When NOT to Use
Incorrect
  • Don't use for sequential workflows (use a Stepper instead)
  • Don't use when all content must be visible at once
  • Don't exceed 8 tabs — use secondary navigation or grouping
  • Don't put tabs inside modals unless absolutely necessary
  • Don't use long labels — keep to 1-2 words in title case

State Appearance Behavior
Active #2196F3 label, medium weight (500), 2px blue indicator at top Shows selected content panel

recommend Best Practices

check_circle Do
Correct
Content panel for selected tab
  • Use for parallel content sections where only one is visible at a time
  • Keep tab labels short (1-2 words) in title case
  • Use scroll variants when tab count is variable or context-dependent
  • Place tabs at the top of the content area they control
  • Use icons only when they add clarity to the label
cancel Don't
Incorrect
  • Don't use for sequential workflows — use a Stepper instead
  • Don't exceed 8 tabs — cognitive load increases significantly
  • Don't put tabs inside modals unless absolutely necessary
  • Don't use for content that must all be seen together
  • Don't use long, descriptive tab labels — keep them scannable
