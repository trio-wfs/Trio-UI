ContentContainer

The standard paper surface for TRIO WFS page content. White background (#FFFFFF), 1px border (#E0E0E0), 4px radius. No shadows — ever. Use for every card, panel, and content section on a page.

category Atomic Component verified Production Ready

info Overview

ContentContainer is the building block for all page content. It provides a consistent white surface with a gray header bar, 1px border, and 4px radius. Containers can nest inside other containers for grouped content sections.

title With Title

Pass a title prop to render the gray header bar (48px, #F5F5F5) with Subtitle2 typography. Optionally pass titleActions to render controls (icons, menus) on the right side of the header.

crop_free Without Title

Omit title to render a borderless surface with only padding — no header bar. Use when the context makes the container's purpose self-evident.

fullscreen Flush Content (padding: 0)

Set padding={0} for content that fills edge-to-edge, like AG Grid. The title bar (if present) still renders normally — only the body padding is removed.

layers Flush Border Radius

Set flush={true} to remove border radius. Use when containers stack directly against each other or against a page edge and rounded corners would create visible gaps.

account_tree Nested Containers

ContentContainers can nest inside each other for grouped sub-sections. Use 16px gap between siblings inside a parent container's padded body.
