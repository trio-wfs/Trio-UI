Viewport rules:

  • 1440px — design target. All layouts built for this width.
  • 1724px — optimal experience. Designs should hold up here.
  • Below 1440px — compromised view. Acceptable but not the priority. Grids scroll horizontally.
  • Mobile — not considered unless explicitly asked. Desktop-first, always.
  • Grid layout — 12-column, 16px gutters. MUI Grid or Box with flex.

layers Content Surfaces — Modal, Panel, Inline

Three ways to show secondary content. The choice depends on whether the user is completing an action, referencing information, or optionally exploring deeper.

Modal (current standard)
Interrupts workflow. User must complete or dismiss before continuing.

Use for: create, edit, save, configure, destructive confirmations, reviewing snapshots.

Sizes: 900px (complex) or 500px (confirmations).

If too complex for 900px → break into steps, don't make the modal bigger.
Panel (newer direction)
Non-interrupting. Main view stays visible and interactive.

Use for: referencing detail info, reviewing context, quick-reference data.

Layout-driven — uses 12-column grid splits (e.g. 8+4). Not a fixed-width component.

Still being developed. Digital Workers is the testing ground.
Inline Expansion
In-place reveal. No overlay, no interruption.

Use for: secondary content the user might want but doesn't need by default.

Collapse/expand is user-driven — don't auto-expand.

Content is subordinate to what's around it.

Future direction: Moving away from modals toward contextual experiences — panels and inline expansion. Content comes to the user in-context rather than interrupting. Until those patterns are proven, modals remain the default for all existing TRIO projects.
