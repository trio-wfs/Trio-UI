Paper

Static surface primitive. Five layered tiers, border-stroke only — no shadows at rest. Hover-shadow behavior lives on interactive wrappers, not on Paper itself.

category Foundation Primitive verified Production Ready

info Overview

psychology Purpose
Foundational surface primitive. Five level values cover the full hierarchy from outermost page canvas to emphasized accent. The base building block for Card, Modal, and most container components.
priority_high Visual Hierarchy
Background — Paper is the canvas, never the focal point. It frames content by establishing a surface tier without competing with what sits on it.
accessibility Accessibility
Plain div semantics by default. Pass role and aria-label when Paper carries meaning (e.g. role="region" for landmark areas).

rule Usage Guidelines

check_circle When to Use
  • Any boxed content surface — panels, sidebars, container regions
  • Pure content layouts where Card's slot structure would be too prescriptive
  • Nested containers (use level="subtle" for the inner)
  • Highlighted/pinned content blocks (use level="accent")
  • As the foundation when building a new composition primitive
cancel When NOT to Use
  • For semantic feedback — use <Alert> for warning/error/success surfaces
  • For headers + bodies + footers — use <Card> for the slot structure
  • For metric tiles — use <MetricCard>
  • As a clickable target — wrap Paper in an interactive component instead
  • As the only differentiator between sections — use spacing first, surface tiers second

toggle_on States

Paper is intentionally non-interactive. There is no hover, focus, active, or disabled state by design. If a surface needs hover-shadow feedback (e.g. clickable cards in a grid), the interactive parent wrapper owns the shadow — Paper stays static.

biotech Anatomy — Subtle in Context

The subtle tier is a 50% white overlay — a de-emphasized substitute for paper. It lifts a canvas surface halfway toward paper without committing to full paper emphasis. Reach for it when you'd otherwise be tempted to drop a card's border to soften it: subtle keeps the border + shape, halves the fill emphasis.

Parent: default canvas (#F5F5F5)
Subtle here — renders as ~#FAFAFA (essentially equal to the secondary surface)
Parent: secondary canvas (#FAFAFA) — primary use case
Subtle here — renders as ~#FCFCFC (clearly distinct from canvas, quieter than paper)
Parent: paper (#FFFFFF) — don't do this
Subtle here — renders as #FFFFFF (the math erases the overlay; no visual effect)

recommend Best Practices

check_circle Do
  • Paper surfaces are border-only, never shadowed at rest — 1px solid #E0E0E0 separates the surface from its parent. Shadows are reserved for floating overlays (modals, menus, popovers, tooltips).
  • For page-level paper sections, use ContentContainer (which composes Paper) — Paper directly is for layout primitives where ContentContainer's title strip would be wrong.
  • Paper levels map to a strict hierarchy: paper (#FFFFFF) for components, secondary (#FAFAFA) for the page content wrapper, default (#F5F5F5) reserved for page canvas only. Don't reach across tiers.
  • Use level="paper" as the workhorse for cards, panels, containers
  • Use level="secondary" for content wrappers below page headers
  • Use level="subtle" as a de-emphasized substitute for paper — secondary metadata cards, stat panels, low-priority summary regions that shouldn't compete with primary content. Only meaningful on the default or secondary canvas.
  • Use level="accent" sparingly for highlighted/pinned content
cancel Don't
  • Don't add box-shadow to Paper at rest — surfaces are border-only in TRIO. Elevation is reserved for floating overlays (Modal, Menu, PopOver, Tooltip, breadcrumb dropdowns).
  • Don't use background.default (#F5F5F5) as a Paper level — that color is reserved for the page canvas, never for a surface. Same for background.secondary (#FAFAFA) outside the content wrapper.
  • Don't use Paper for semantic feedback (warning, error, success) — use <Alert> with the appropriate severity.
  • Don't stack multiple accent surfaces — the highlight loses meaning
  • Don't use Paper as the only differentiator between sections — start with spacing, then surface tier
  • Don't use level="subtle" on a paper parent — the 50% white overlay has no visual effect on #FFFFFF. If you need de-emphasis inside paper, drop the border or use a flat region instead.
