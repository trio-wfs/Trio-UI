Avatar

Identity primitive for user and agency authors. The anonymized variant powers cross-agency privacy in shared threads.

category Atomic Component verified Production Ready

info Overview

psychology Purpose
Circular identity primitive rendering 1–2 initials over a colored fill. Anchors authorship on posts, comments, and assignee chips throughout TRIO.
priority_high Visual Hierarchy
Low — Avatar is a supporting identifier, never the primary action on a row. It scans as a colored dot first, identity second.
accessibility Accessibility
Renders as role="img" with an aria-label derived from the initials. Non-interactive — no keyboard focus or hover state.

rule Usage Guidelines

check_circle When to Use
  • Identifying authors on posts, comments, and Q&A threads
  • Showing the assignee on a worker, shift, or order row
  • Visualizing privacy mode (other agencies as anonymized grey)
  • Stacked group avatars for collaborators on a record
  • TRIO Staff identification — slate Avatar is the persistent staff anchor
cancel When NOT to Use
  • As an interactive control — wrap in a button or link instead
  • For displaying photos — Avatar is initials-only by design
  • For status indicators — use Badge or Chip
  • For program/facility icons — those have a different visual treatment
  • At sizes other than 24px/32px — extend the variant set in Figma first

toggle_on States

Avatar is intentionally non-interactive. There is no hover, focus, active, or disabled state by design — Avatar is a display primitive. When the surrounding context (a row, card, or button) needs interactivity, wrap the Avatar in that interactive container instead of adding state behavior here.

recommend Best Practices

check_circle Do
  • Avatar is non-interactive chrome — it identifies the author, it isn't clickable on its own. Wrap in an interactive parent if the surrounding row needs click behavior.
  • The Avatar color palette is closed — staff, anonymized, plus 6 hash colors. Don't extend the palette ad-hoc; if a new role needs an avatar color, add it in Figma first.
  • Use color="staff" for every TRIO Staff / RM author — slate anchors staff identity across all threads
  • Hash agency identifier colors from a stable input (e.g. agency name) so the same author keeps the same color across sessions
  • Use color="anonymized" in cross-agency views — uniform grey is the privacy contract made visible
  • Pass exactly 1–2 character initials; truncate before passing if the source is longer
cancel Don't
  • Don't add hover or click behavior to Avatar itself — Avatar is identity chrome, not a control. If the row is clickable, the parent owns the interaction.
  • Don't invent additional sizes or colors — the variant set is locked. Extend in Figma first so the design system stays the source of truth.
  • Don't pass staff or anonymized through the hash function — both are reserved for specific roles
  • Don't pass more than 2 characters — they will overflow the circle
  • Don't add a drop shadow at rest — Avatar is a paper surface, border-only
