Footer

The application footer that sits at the bottom of every TRIO WFS page. Contains copyright information and legal links.

info Overview

A simple, single-row footer. Copyright text is left-aligned, legal links are right-aligned and separated by a pipe character. The footer spans the full width of the page and uses the page canvas background color.

view_agenda Page Placement

The footer always sits at the absolute bottom of the page, below all content. It is not sticky — it scrolls with the content. In the page layer stack it sits outside the content wrapper, directly on the page canvas.

recommend Best Practices

check_circle Do
  • The footer is sticky and appears on every page — 16px horizontal padding, 16px vertical padding on copy, 1px #E0E0E0 top border (the standard component border, not the input border).
  • Footer links use primary blue (#2196F3) — one of the few sanctioned uses of primary outside Save/Update actions, since clickable text is the exception.
  • Dynamically generate the copyright year and version number
  • Internal links open in the same tab; external links open in a new tab
  • Use caption (12px) typography — the footer is supporting chrome, not primary content
cancel Don't
  • Don't add a drop shadow to the footer — shadows are reserved for floating overlays (modals, menus, popovers). The footer is a paper surface and uses the border, not elevation.
  • Don't hardcode the copyright year or version number — they should be derived from build/runtime data
  • Don't elevate footer typography above caption (12px) — body or larger text competes with primary page content
