Navigation Vertical

The primary left-sidebar navigation used across all TRIO WFS desktop views.

category Custom Component menu_book Storybook design_services Zeplin verified Production Ready

info Overview

psychology Purpose
NavigationVertical is the persistent left-rail navigation. It holds primary section links with icons, supports expandable sub-navigation, and includes an optional settings section at the bottom.
priority_high Layout
Width is fixed at 260px. Primary items use icon + label + optional chevron. Settings section uses a distinct left-border selected style instead of background fill.
accessibility Accessibility
All nav items are keyboard-navigable buttons. Expandable items toggle aria-expanded. Settings items are visually differentiated from primary nav for quick scanning.

rule Usage Guidelines

check_circle When to Use
Correct
Icon + short label for every primary item
  • Always include a left icon for primary nav items — critical for scanability
  • Keep primary nav labels short — 1 or 2 words maximum
  • Auto-expand the active parent item on page load
  • Use settings section for configuration pages only
cancel When NOT to Use
Incorrect
Nested 3+ levels deep
  • Don't use settings-style items (left border) for primary navigation
  • Don't nest sub-navigation more than one level deep
  • Don't put more than 8–10 primary items in the nav
  • Don't use for mobile — this is desktop-only

recommend Best Practices

check_circle Do
  • Always include a left icon for primary nav items — icons are critical for scanability in time-pressured clinical environments
  • Keep primary nav labels short — 1 or 2 words maximum. The 260px width is tight with an icon, label, and chevron
  • Auto-expand the active parent item on page load so users can see where they are in the hierarchy immediately
cancel Don't
  • Don't use settings-style items (left border) for primary navigation — the two patterns are visually distinct for a reason
  • Don't nest sub-navigation more than one level deep — if you need deeper hierarchy, reconsider the IA
  • Don't put more than 8–10 primary items in the nav — consider grouping or a secondary navigation pattern
