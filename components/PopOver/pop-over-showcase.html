PopOver

An anchored floating panel that presents contextual content next to a trigger element — used for previews, secondary details, or quick actions that don't warrant a full Modal.

category Custom Component verified Production Ready

info Overview

psychology Purpose
Floating panel anchored to a trigger, dismissed by outside click or Escape. Used for option panels, filter pickers, info bubbles, and lightweight interactive surfaces that don't warrant a full modal.
priority_high Visual Hierarchy
Medium when open — Pop-over draws focus while displayed but does not block the rest of the page.
accessibility Accessibility
Focus moves into the pop-over on open. Escape closes and returns focus to the trigger. Click-outside dismissal must be paired with a keyboard equivalent.

rule Usage Guidelines

check_circle When to Use
  • Surface a small amount of secondary context next to a trigger (worker notes, status timeline, recent activity)
  • Offer a focused, scoped action set tied to a specific row, card, or element
  • Show a list or summary that's too rich for a Tooltip but too lightweight for a Modal
  • Provide an inline "view all" handoff via the optional header action link
cancel When NOT to Use
  • Don't use for blocking flows — use Modal when the user must respond before continuing
  • Don't use for forms with more than ~3 inputs — promote to Modal or a side Panel
  • Don't use for tiny hover hints — use Tooltip
  • Don't overlap two PopOvers — only one anchored panel should be open at a time

recommend Best Practices

check_circle Do
Correct
Short title · scoped content · clear "View all" handoff
  • PopOver is a floating overlay — drop shadow is allowed (along with Tooltip, Menu, and Modal). This is the sanctioned use of elevation; everywhere else in the system surfaces use borders, not shadows.
  • PopOver is for preview / scoped content with an interactive escape hatch — that's the distinction from Tooltip (passive, hover-only, dismisses on move-away) and Modal (blocking, full attention required).
  • Anchor the popover next to its trigger so the relationship is obvious
  • Use a short, descriptive title (1–3 words) — match the trigger label's intent
  • Provide a "View all" or "Open …" action link when the content is a preview of a larger view
  • Use the default 450px width for most cases; only override when the content is genuinely narrower
cancel Don't
Incorrect
Forms, blocking actions, multi-section content
  • Don't put long forms in a PopOver — promote to Modal (default for committing actions in TRIO) or a side Panel for review/reference.
  • Don't use PopOver for destructive confirmations — the dismiss-on-outside-click contract means the user can lose the action accidentally. Modal carries the right weight.
  • Don't stack PopOvers (one opens another) — only one anchored panel should be open
  • Don't rely on hover to open — clicks are the contract; hover belongs to Tooltip
