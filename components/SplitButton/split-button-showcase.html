Split Button

A two-segment button: the left triggers a primary action, the right opens a dropdown menu of secondary options.

category Custom Component verified Production Ready

rule Usage Guidelines

check_circle When to Use
Correct
  • One clear primary action plus 2–5 related secondary actions
  • Save/update flows where the main action is obvious
  • Export scenarios with multiple format options
cancel When NOT to Use
Incorrect
  • Don't use if all options are equally weighted — use Menu or ButtonGroup
  • Don't use if there is only one action — use a regular Button
  • Don't stack multiple split buttons next to each other

widgets Variants

Primary
Blue fill · white text · white divider — use for save/update actions
Secondary
Gray fill · gray border · dark text — use for edit/utility actions

format_paint Fill Style

Contained
Solid fill — use for the primary action on a page
Outline
Bordered, transparent fill — use for secondary emphasis or lighter surfaces

toggle_on States

Default
Open
Arrow rotates 180° when menu is open
Disabled
Interactive Demo — click the arrow to open the dropdown

recommend Best Practices

check_circle Do
Correct
  • Label the button with the primary action — "Save", "Submit", "Export"
  • Keep menu options closely related to the primary action
  • Use primary color for save/update flows, secondary for utility actions
  • Limit menu to 3–5 options — if more, consider a separate Menu
cancel Don't
Incorrect
  • Don't use as a replacement for a plain dropdown — the label must trigger something
  • Don't use for destructive actions without confirmation in the menu item
  • Don't place two split buttons adjacent to each other
  • Don't use in mobile contexts — targets are too small
