Menu

Displays a list of choices on a temporary surface. Use when you need to provide users with multiple options or actions in a compact, contextual way.

category Navigation verified Production Ready

ads_click Trigger Patterns

The Menu is a headless surface — it doesn't include a trigger element. It anchors to whatever element opens it. These are the three primary trigger patterns used across TRIO WFS.

smart_button Button Trigger

Use with Chip for filter bars. Clicking a chip opens a multi-select menu. The chip updates to show active state when filters are applied. Common in grid toolbars.
info
Implementation note: The Menu component accepts anchorEl and open props. Store a ref to the trigger element and pass it as anchorEl when the menu should open. The menu positions itself automatically relative to the anchor. Use state="multi" for chip filter menus where multiple selections are allowed.
