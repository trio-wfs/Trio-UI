Badge

Badges are small status indicators anchored to an element — used to show counts, notifications, or state without interrupting the user's flow.

infoOverview

Use the default type to show numeric counts (unread messages, pending items). Use the dot type as a simple presence indicator when the count isn't meaningful. In VMS workflows, badges commonly appear on nav items, icons, and list items to signal pending credentialing, timecard, or submission activity.

ruleBest Practices

Do
  • Use error color for expired credentials, overdue timecards
  • Use primary for general counts (pending submissions, new messages)
  • Cap displayed counts at 99+ for large numbers
  • Use dot when the user needs to notice activity, not count it
Don't
  • Don't use badges for decorative purposes
  • Don't place a badge on text — anchor to icons or avatar elements
  • Don't mix badge colors arbitrarily — color must communicate severity

color_lensColor Semantics

General activity counts — new submissions, unread messages.
Error
Urgent items requiring action — expired documents, overdue timecards.

Items approaching a deadline or threshold.
Success
Completed or verified items — use sparingly.
Info
Neutral informational counts.
Secondary
De-emphasized counts — inactive or archived items.
