Overview

info Purpose
Menus display a list of choices on temporary surfaces. Use when:
  • Showing contextual actions or options
  • Providing navigation in a compact space
  • Organizing related actions together
tune States
Menu variants: single, multi, grouped
Item states: default, selected, disabled, title
Min width: 254px
Item height: 37px
Padding: 16px horizontal, 8px vertical
accessibility Accessibility
  • Keyboard navigation (Arrow keys, Enter, Escape)
  • Focus management (returns to trigger on close)
  • ARIA roles and labels
  • Screen reader announcements

rule Usage Guidelines

check_circle When to Use
  • Group related menu items together
  • Use dividers to separate logical groups
  • Keep menu item labels concise and clear
  • Position menu near the trigger element
  • Use icons for common actions (copy, delete, etc.)
cancel When NOT to Use
  • Don't use menus for critical actions without confirmation
  • Don't exceed 10-12 items without grouping or scrolling
  • Don't use menus for long-form content
  • Don't nest menus more than one level deep
  • Don't hide primary actions in menus
