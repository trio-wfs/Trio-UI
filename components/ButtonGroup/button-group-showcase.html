Button Group

Groups related button actions together to show multiple options in a single, connected unit. Use to present related actions that should be visually grouped.

category Atomic Component menu_book Storybook design_services Zeplin verified Production Ready

info Overview

psychology Purpose
Button groups connect related actions to show they belong together. Use when multiple actions modify the same object, are related, or when you need to save horizontal space for multiple buttons.
tune Variants
Variant: contained, outline
Size: sm (32px), md (38px)
Color: primary, secondary
Orientation: horizontal, vertical
Buttons: 2-6 buttons
accessibility Accessibility
Each button is individually focusable via keyboard. Use clear, descriptive labels for each button. Proper ARIA roles are handled by MUI ButtonGroup.

rule Usage Guidelines

check_circle When to Use
Correct
  • Group related actions that modify the same object
  • Use 2-4 buttons for optimal usability
  • Use clear, action-oriented labels
  • Use primary color only for confirming/save actions
  • Use secondary for editing/prompt actions
cancel When NOT to Use
Incorrect
  • Don't group unrelated actions together
  • Don't exceed 6 buttons (Figma limit: button-8)
  • Don't use for exclusive selection (use Toggle Button)
  • Don't mix primary and secondary colors in one group
  • Don't use for navigation (use Tabs instead)

widgets Variants

Button groups come in two sizes and two color schemes. Primary blue is for confirming/save actions. Secondary is for editing/prompt actions.

Small (32px) - Toolbars and dense interfaces
Medium (38px) - Standard forms and modals
Primary - Confirming Actions
Secondary - Editing/Prompt Actions
Vertical - For narrow containers

toggle_on States

Button group states mirror individual button states. Hover any button in the group to see the interactive state.

  • Use primary for save/confirm actions only
