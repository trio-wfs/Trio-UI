Page Header Toolbar

The top-of-page toolbar used across all TRIO WFS list and detail views. Contains the page title, optional chips and eyebrow, right-side actions, and an optional Breadcrumb strip.

category Custom Component menu_book Storybook design_services Zeplin verified Production Ready

info Overview

psychology Purpose
The Page Header Toolbar is a white card that anchors every page. It provides title, context (eyebrow, chips), and page-level actions in a consistent layout across all views.
priority_high Variants
The full variant includes title, chips, eyebrow, actions, and breadcrumb strip. The default strips it back to title + buttons only. NewCanvas removes the outer card border.
accessibility Accessibility
Uses semantic header elements and ARIA labels. Breadcrumb links are keyboard-navigable. Action buttons follow the standard focus ring pattern.

rule Usage Guidelines

check_circle When to Use
Correct
Title + Actions + Breadcrumb on every list/detail page
  • Use full as the default for list/grid views — breadcrumb + chips provide important context
  • Use default for simple utility pages without deep navigation
  • Keep page title to the entity name — avoid verbs (use "Shift Scheduler", not "Manage Shifts")
  • Put the primary action in ButtonGroup, secondary in SingleButton
  • Use indicator only when the entity has a meaningful status affecting the whole page
cancel When NOT to Use
Incorrect
Multiple toolbars or destructive actions in toolbar
  • Don't put more than 4 chips in the title row — truncate with "+N More"
  • Don't use more than one toolbar per page
  • Don't put destructive actions (Delete, Archive) in the toolbar — move to a confirmation modal flow
  • Don't omit eyebrow text on complex pages — it provides orientation for new users
