Select

Allows users to choose a single option from a dropdown list. Use when you have 5 or more options that would clutter the interface as radio buttons.

category Form Control menu_book Storybook design_services Zeplin verified Production Ready

info Overview

psychology Purpose
Select dropdowns let users choose one option from many. Use when you have 5 or more options, space is limited, or options are well-known (countries, states, etc.).
tune Anatomy
Built from a TextField input with a dropdown icon and a custom Menu component. States include default, focus, disabled, error, and selected. Input height: 36px, min width: 256px.
accessibility Accessibility
Keyboard accessible with Arrow keys to navigate and Enter to select. Always include a visible label and use helper text for additional context. Provide clear error messages for validation.

rule Usage Guidelines

check_circle When to Use
Correct
High arrow_drop_down
  • Use for 5+ options (otherwise use Radio buttons)
  • Sort options logically (alphabetically or by usage)
  • Provide a default selection when appropriate
  • Use clear, concise option labels
  • Include helper text for complex selections
cancel When NOT to Use
Incorrect
Select arrow_drop_down
  • Don't use for 2-4 options (use Radio buttons instead)
  • Don't use for actions (use Button or Menu instead)
  • Don't use overly long option names
  • Don't change content based on selection without user action
  • Don't nest select dropdowns
