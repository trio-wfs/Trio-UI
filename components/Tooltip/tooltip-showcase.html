Tooltip

A short contextual label that appears on hover to clarify the purpose of a UI element.

info Overview

Tooltips display a brief text label when a user hovers over or focuses an element. They are always black with white text, and include a directional arrow pointing at the trigger. Use tooltips to clarify icon-only buttons, truncated text, or secondary actions where adding a permanent label would clutter the UI.

thumb_up Best Practices

✓ Do

Keep tooltip text short — one phrase or sentence. Tooltips are supplemental, not primary documentation.

✗ Don't

Don't put interactive content (links, buttons) inside a tooltip. Tooltips dismiss on mouse leave and aren't accessible for click.

✓ Do

Use tooltips on icon-only ButtonIcon components to always provide an accessible text label for screen readers.

✗ Don't

Don't use tooltips to surface critical information that users must see to complete a task. Use inline help text or an Alert instead.

✓ Do

Choose the position that keeps the tooltip visible within the viewport — prefer top or bottom for most UI contexts.

✗ Don't

Don't add tooltips to disabled elements — users can't trigger them and the tooltip may mislead. Use inline disabled messaging instead.
