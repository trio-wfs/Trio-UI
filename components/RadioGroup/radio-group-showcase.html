Radio Group

Allows users to select exactly one option from a set. Use when users must choose a single option and all options should be visible at once.

info Overview

psychology Purpose
Radio buttons let users select exactly one option from a set. Use when users must choose one option (mutually exclusive), all options should be visible (2-5 options), and comparing options is important.
tune Anatomy
Each radio has a 38x38px hitbox with a 16px visible circle. States include checked/unchecked, disabled, and error. Colors: primary (#2196F3), default (gray), error (#DB4537). Hover shows a circular ripple.
accessibility Accessibility
Keyboard accessible with Arrow keys to navigate and Space to select. Group with fieldset/legend for screen readers. Always pre-select one option by default. Use clear, distinct labels for each option.

rule Usage Guidelines

check_circle When to Use
Correct
  • Use for mutually exclusive selections
  • Display 2-5 options (use Select for more)
  • Pre-select the most common/safe option
  • Stack vertically for easy scanning
  • Use clear, concise labels
cancel When NOT to Use
Incorrect
  • Don't use for multiple selections (use Checkbox instead)
  • Don't use for more than 5 options (use Select instead)
  • Don't use for actions (use Button instead)
  • Don't leave unselected if one option must be chosen
  • Don't use for binary yes/no choices (use Checkbox or Switch)

recommend Best Practices

check_circle Do
Correct
  • Always pre-select the most common or safest option
  • Stack options vertically for easier scanning
  • Keep labels short and descriptive
  • Group related radio buttons with a clear heading
  • Use for single-choice selections with 2-5 visible options
cancel Don't
Incorrect
  • Don't use radio buttons for multi-select — use Checkbox
  • Don't use more than 5 options — switch to Select dropdown
  • Don't leave the group without a default selection
  • Don't place radios horizontally if labels are long
  • Don't use for on/off toggles — use Switch instead
