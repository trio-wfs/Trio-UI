Breadcrumb

Navigation path strip showing the user's location within a hierarchy. Sits between the Page Header Toolbar and the main content area. Supports an optional record count and action buttons on the right.

category Atomic Component menu_book Storybook design_services Zeplin verified Production Ready

info Overview

psychology Purpose
The Breadcrumb is a 40px-tall strip with a light gray background. It renders an ordered list of navigation links showing the user's current location within the application hierarchy.
tune Configuration
Two divider styles — breadcrumb uses / and Links uses |. The right side optionally shows a record count and a ToggleButton strip for filter/export/reset actions.
accessibility Accessibility
The current page is displayed in dark text with a blue bottom underline, clearly distinguishing it from clickable ancestor links. Uses semantic navigation landmarks.

rule Usage Guidelines

check_circle When to Use
Correct
  • Showing hierarchical navigation within the application
  • Pages that are 2+ levels deep in the information architecture
  • List or table views where a record count is helpful
  • Pages with filter/export/reset toolbar actions
cancel When NOT to Use
Incorrect
  • Don't exceed 6-7 path segments — truncate middle items with ellipsis
  • Don't use for step-by-step wizard navigation — use a Stepper
  • Don't put non-navigation actions in the breadcrumb strip
  • Don't use on top-level pages with no parent hierarchy

widgets Variants

Two divider styles — breadcrumb uses /, Links uses |. Use breadcrumb as the default; Links for tab-style navigation.

toggle_on States

Record count and actions slot are independently toggleable. The breadcrumb adapts to show only the elements that are enabled.

recommend Best Practices

check_circle Do
Correct
  • Always mark exactly one item as selected — the current page
  • Keep link labels short — use the page/section name only
  • Use the / state as default; | for tab-style navigation
  • Show record count when the page is a list or table view
cancel Don't
Incorrect
  • Don't exceed 6-7 path segments — truncate middle items with ellipsis
  • Don't put non-navigation actions in the breadcrumb strip
  • Don't use breadcrumb for step-by-step wizard navigation — use a Stepper
  • Don't use single-letter or ambiguous labels
