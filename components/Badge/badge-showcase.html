Badge

Badges are small status indicators anchored to an element — used to show counts, notifications, or state without interrupting the user's flow.

category Atomic Component verified Production Ready

info Overview

psychology Purpose
Small overlay marker for unread counts, status indicators, or notifications. Anchored to a parent element — typically an icon, avatar, or button.
priority_high Visual Hierarchy
Low — Badge is a visual ornament that supplements the element it's attached to. Never the primary content.
accessibility Accessibility
The visual mark is decorative. The parent element must carry an aria-label that communicates the count or status (e.g., "Inbox, 5 unread").

rule Usage Guidelines

check_circle When to Use
Correct
  • Use error color for expired credentials, overdue timecards
  • Use primary for general counts (pending submissions, new messages)
  • Cap displayed counts at 99+ for large numbers
  • Use dot when the user needs to notice activity, not count it
  • Anchor badges to icons or avatar elements
cancel When NOT to Use
Incorrect
Total Items
  • Don't use badges for decorative purposes
  • Don't place a badge on text — anchor to icons or avatar elements
  • Don't mix badge colors arbitrarily — color must communicate severity
  • Don't use badges as standalone status labels (use Chip instead)

palette Color Semantics

Badge colors communicate severity and type of pending activity in VMS workflows.

Color When to Use Example Visual
primary General activity counts New submissions, unread messages
error Urgent items requiring action Expired documents, overdue timecards
warning Items approaching a deadline Credentials expiring soon
success Completed or verified items Approved timecards — use sparingly
info Neutral informational counts Available shifts, total open orders
secondary De-emphasized counts Inactive or archived items
