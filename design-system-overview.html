stars Design Principles

Our design system is built on core principles that guide every decision, from component design to implementation patterns.

verified Figma as Source of Truth
Every component is built by extracting actual values from Figma component instances. No assumptions, no interpretations—just pixel-perfect accuracy. Design tokens define what exists; Figma instances define how to apply them.
desktop_windows Desktop-First
Designed exclusively for desktop SaaS applications. No responsive breakpoints, no mobile considerations. This focused approach allows for optimal desktop experiences without compromise.
accessibility Accessibility Built-In
All components meet WCAG 2.1 AA standards with proper focus states, keyboard navigation, screen reader support, and minimum touch targets of 32px.
speed Consistency & Efficiency
Standardized patterns reduce decision fatigue. The 8px spacing system, semantic color rules, and predictable component behavior create a cohesive experience across all products.

rule Core Rules

Spacing System
8px base grid for all layout spacing. Exception: 4px for related content within components. Never break the grid for arbitrary values.
Typography
Roboto only, max H5 for page headers. Font sizes from 11px-24px scale. Desktop-optimized line heights.
Color Semantics
Primary blue (#2196F3) for Save/Update only. Secondary for Edit. Error for Delete. Info for read-only. Background (#F5F5F5) for canvas, Paper (white) for containers.
Icons
Material Icons only. No custom icon sets. Icons reinforce action meaning but should not replace clear text labels.
Data Tables
AG Grid for all tables. No custom table implementations. Consistent sorting, filtering, and pagination across products.
Layout
12-column flex grid with 16px margins and gutters. Modals use 6-8 columns with 40px gutters and 16px margins.

build Implementation Workflow

Every component follows a mandatory 4-step extraction workflow to ensure Figma accuracy.

Step 1: Extract Properties
Read componentPropertyDefinitions from cached Figma spec. Extract VARIANT, BOOLEAN, and TEXT properties. Map to TypeScript interfaces.
Step 2: Load Design Tokens
Import design tokens to understand available colors, spacing values, typography options, and shapes.
Step 3: Extract Actual Values
CRITICAL: Extract padding, dimensions, fills, strokes from actual Figma component instances. Never assume based on tokens.
Step 4: Implement
Build component using extracted values. Reference design tokens for colors/typography, but use actual extracted measurements.
