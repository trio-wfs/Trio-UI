Alert

Alerts communicate important messages to users — informational, success, warning, or error — with optional actions and dismiss controls.

infoOverview

Use alerts to surface system messages, validation results, or status updates. Always pair with a clear title and concise description. In clinical workflows, alert severity directly communicates patient safety risk — use error and warning sparingly and intentionally.

ruleBest Practices

Do
  • Use error only for blocking issues that require immediate action
  • Keep titles short — 4 words or fewer
  • Pair every alert with a clear action or dismiss path
  • Use standard variant as the default — it's the least visually disruptive
Don't
  • Don't use contained for low-priority messages — it demands full attention
  • Don't stack multiple alerts of the same severity
  • Don't use alerts for empty states or loading states
  • Don't rely on color alone — always include an icon and text

color_lensColor Semantics

Default (Blue)
Informational messages. No action required. Use for system status, tips, or guidance.
Error (Red)
Something failed or is blocked. Requires user action before proceeding. Use sparingly.
Warning (Orange)
Potential issue — non-blocking. User should be aware but can proceed. Use for expiring credentials, approaching limits.
Success (Green)
Positive confirmation. Action completed successfully. Auto-dismiss after 3–5 seconds when possible.
